# Threat & Vulnerability Monitoring Procedure

## 1. Monitoring Sources
- CVE database
- Automotive ISAC
- Internal ECU and gateway logs
- security_monitor.log
- attacker_node test results

## 2. Monitoring Activities
- Daily review of CVEs affecting automotive components
- Weekly review of ISAC advisories
- Continuous log collection from ECUs and gateways
- Automated anomaly detection via monitoring simulator

## 3. Reporting
- Findings documented in Threat & Vulnerability Register
- Critical findings escalated to Cybersecurity Officer

## 4. Evidence Storage
- Logs stored under logs/security_monitor.log
- Snapshots stored under compliance/R155/CSMS/snapshots
